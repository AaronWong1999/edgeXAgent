edgeX Agent — Navigation Blueprint v2

Restructured: 2 levels + push layer. Every screen branded. Single-message navigation.

Architecture: 2 Levels + Push Layer

  • L1 — Dashboard: /start sends new message. 7 buttons. Root of all navigation.
  • L2 — Feature Screens: All edit the dashboard message in-place. Every screen has {ICON} Title — edgeX Agent title + 🔙 Back.
  • L3 — Sub-screens: Deeper settings (News Freq, Add Source, Memory, Clear Confirm). Edit same message. Back → parent L2 → Dashboard.
  • Push — Bot-Initiated: News alerts, AI chat replies, trade plans, trade results. Always new messages (separate from dashboard). Buttons edit the push message in-place (confirm → executing → result).

Push Flow 1: News Alert

📰 News Push (new msg to user)
  ├── 📈 LONG $50/$150 → Trade Plan (new msg) → ✅ Confirm → Executing... (edits plan) → Result (edits again)
  ├── ❌ Cancel → Cancelled (edits plan msg, shows Status/P&L/Menu buttons)
  ├── 🇨🇳🇯🇵🇰🇷🇷🇺 → Translated Card (new msg, same trade buttons)
  ├── 🔗 Source → opens URL
  ├── 🔕 Mute → toast "Source muted", removes buttons
  └── ❌ Dismiss → deletes message

Push Flow 2: AI Chat

User types message → 💬 AI Reply (new msg)
  ├── CHAT: text + 🏠 Main Menu
  ├── TRADE: Trade Plan with ✅ Confirm | ❌ Cancel
  │   ├── ✅ Confirm → Executing... (edits) → Result (edits same msg again)
  │   │   Result shows: Status | P&L | Close {asset} | History | 🏠 Main Menu
  │   └── ❌ Cancel → Cancelled (edits, shows Status/P&L/Menu)
  └── NO EDGEX: Plan + 🔗 Connect edgeX to Execute
